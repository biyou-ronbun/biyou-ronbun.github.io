# サイトの使い方

Node も git も不要です。PowerShell だけで動きます。

---

## 1. サイトを作り直す（ビルド）

記事を追加・修正したら、これを1回実行するだけです。

```powershell
powershell -ExecutionPolicy Bypass -File site\build.ps1
```

`site\dist\` に、そのままアップロードできる完成品のサイトが出力されます。

確認するには `site\dist\index.html` をダブルクリックしてブラウザで開いてください。

---

## 2. 記事を追加する（＝自動掲載の仕組み）

**手順は2つだけです。**

### ① 記事の Markdown を置く

`articles\<スラッグ>.md` に置きます。いつもどおり writer が作ってくれるものです。

### ② `site\articles.json` に1ブロック足す

```json
{
  "slug": "turnover-28days",
  "title": "「肌のターンオーバー28日周期」の出典はどこにあるのか",
  "subtitle": "美容界で最も引用される数字を、遡ってみる",
  "date": "2026-07-20",
  "category": "習慣",
  "summary": "カードやトップページに出る、2〜3行の要約。",
  "published": true
}
```

あとはビルドするだけ。トップページ・記事ページ・RSS・サイトマップが**全部自動で更新されます。**

| 項目 | 意味 |
|---|---|
| `slug` | `articles\<slug>.md` と一致させること。URLにもなります |
| `category` | 記事カードに出るタグ。`塗る` / `飲む` / `習慣` など自由 |
| `date` | `YYYY-MM-DD`。並び順と RSS の日付になります |
| `published` | **`false` にすると、書けているのに公開されません。** 記事を寝かせるときに使う |

**`published: false` は「週2本ずつ出す」運用のための機能です。** 記事は全部書き上げておいて、出す分だけ `true` にする。これで在庫を持ちながら、公開ペースをコントロールできます。

---

## 3. サイトを公開する（インターネットに出す）

**Netlify Drop がいちばん簡単です。アカウント登録もコマンドも不要で、無料です。**

1. ブラウザで <https://app.netlify.com/drop> を開く
2. `site\dist` フォルダを、ページの上にドラッグ＆ドロップする
3. 数十秒で `https://ランダムな名前.netlify.app` というURLが発行される。これがあなたのサイトです

更新するときも、同じページに `site\dist` をドロップし直すだけです。

> **公開したら必ずやること**
> `site\config.json` の `baseUrl` を、発行された実際のURLに書き換えてビルドし直してください。
> RSS・サイトマップ・OGP に埋め込まれるURLがここから作られるので、放置すると検索エンジンとSNSに誤ったURLが伝わります。

### 独自ドメインが欲しくなったら

Netlify の管理画面から独自ドメイン（例: `example.com`）を設定できます。ドメインは年1,000〜2,000円ほどで買えます。SSL（https）は Netlify が無料で自動設定します。

### もっと自動化したくなったら

Node と git を入れて GitHub に置けば、「記事を追加して push するだけで、勝手にビルドされて勝手に公開される」ところまで自動化できます。まずは上のドラッグ＆ドロップで運用してみて、面倒だと感じたらで十分です。

---

## 4. 見た目を変える

| 変えたいもの | 触るファイル |
|---|---|
| サイト名、キャッチコピー、免責文、URL | `site\config.json` |
| 色、文字サイズ、余白 | `site\assets\style.css`（先頭の `:root` の色だけ変えれば雰囲気が変わります） |
| ページの構造、ヘッダー、フッター | `site\templates\*.html` |
| 「このブログについて」の文章 | `site\templates\about.html` |

ダークモードは、読者のOS設定に自動で追従します（`style.css` の `prefers-color-scheme`）。

---

## 5. 注意

- **`site\dist\` は毎回まるごと作り直されます。** ここを直接編集しても、次のビルドで消えます。編集するのは `articles\`、`site\config.json`、`site\templates\`、`site\assets\` です。
- **`build.ps1` に日本語を書かないでください。** Windows PowerShell 5.1 が日本語を含むスクリプトを文字化けさせるため、このスクリプトは意図的に英数字だけで書かれています。日本語はすべて JSON とテンプレートに置いてください。
