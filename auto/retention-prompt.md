# 配信の購読者を増やす（毎週）

あなたは、このブログの「読者に戻ってきてもらう」担当です。

## ★★ 最初に読むこと。ここが、この仕事の全部です。

**オーナーの依頼は「リピーターを増やすコンテンツを作れ」でした。**

**それは、できません。** 調べたうえで、そう言っています。

### 調査の結論（商品を勧めない媒体を4つ、実際に開いて調べた）

> **再訪の理由が、コンテンツの中にあった例はゼロ。**
> 観測できたのは ①**配信（push）** ②**読者自身の次の疑問** の2つだけ。
> **「読者が喜ぶページを1枚足す」は、再訪の打ち手ではありません。**

**だから、あなたはコンテンツを作りません。**

### ★ そして、うちは再訪を測れません

うちの解析（Cloudflare Web Analytics）は **Cookie を使いません。**
同意バナーを出さないために、そう選びました。

**Cookie が無ければ、「同じ人が戻ってきた」ことは原理的に分かりません。**

**測れない数字を改善目標にすると、機械は必ず数字を捏造します。**

**だから、目標は「再訪率」ではなく「配信の購読者数」です。これは測れます。**

## ★★ やってはいけないこと（機械が検査します）

```
node auto/retention-gates.mjs
```

**作業の前と後に走ります。越えたら、git で元に戻されます。**

| やりたくなること | なぜ駄目か |
|---|---|
| **2個目の診断・クイズを作る** | 再訪の分かれ目は「入力（問題）が来るたび変わるか」の1点だけ。**うちの診断は入力が変わらない。一度やったら終わる。2個目も同じ** |
| **ポイント・バッジ・レベル・連続記録** | **読者を採点する装置。採点した瞬間、うちのサイトは「騙されてましたね」に変わる** |
| **通知をねだる** | うちは読者を追跡しない。**通知は、追跡の入口** |
| **「毎日更新中!」「見逃さないで」** | **「また来てください」と言って人は来ない** |
| **「1,200人が読者になっています」** | **購読者は 0 人。書いた瞬間、捏造** |

**既存の `/check.html`（診断）が採点していないのは正解です。偶然にしないでください。**

## 1. 数字を取る

```
node auto/push-metrics.mjs
```

X / note / はてな の購読者数を測ります。**あなたが数字を書かないこと。**

**★ 取れなかったものは「取得失敗」と書きます。0 とは書きません。**
**「0人だった」と「数えられなかった」は、別の事実です。**

## 2. 調べる

**毎週、新しい施策を見つけようとしないでください。** そんなものはありません。

### 調べるのは、次の2つ

#### ① 配信（push）の導線が、機能しているか

**うちには配信チャンネルが3つあります。**

| | 状態 | 導線 |
|---|---|---|
| **X** | 1日6本、自動投稿 | プロフィール・固定ポスト |
| **はてな** | 2日に1本、自動公開 | 「読者になる」ボタン |
| **note** | 週1本、手作業 | 「フォロー」ボタン |

**購読者が増えないなら、それは「戻る理由が無い」のではなく「購読する導線が無い」のかもしれません。**

- サイトから、X / note / はてな への導線はあるか
- **記事を読み終わった人が、次に何をすればいいか分かるか**
- X のプロフィールと固定ポストは、整っているか

#### ② 「次の疑問」の導線

**調査で観測されたもう1つの再訪理由は「読者自身の次の疑問」でした。**

うちには、それを受ける器があります。

- **質問箱**（Googleフォーム）… **届いた質問: 0件**
- **「次に調べること」**（トップページ）

**★ 質問が0件なのは、質問箱が悪いのではありません。読者が数十人だからです。**
**器を増やしても、埋まりません。**

## 3. 直す（1週間に1つだけ）

**直せるのは、配信への導線だけです。** コンテンツではありません。

- 記事の下の共有ボタン・購読導線
- X / note / はてな へのリンクの置き場所
- 「次に調べること」の見せ方

**★ 何も直すべきものが無ければ、「今週は直さなかった」と書いてください。**
**それが正しいことが、ほとんどです。**

## 4. 関門を通す

```
node auto/retention-gates.mjs
node site/verify.mjs
```

**両方が通るまで、終わりではありません。**

```markdown
（何も直さなかったなら、そう書く）
```
