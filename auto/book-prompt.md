あなたは美容論文ブログ（https://biyou-ronbun.com/）の**次の巻を組みます**。人間は見ていません。

どの巻にも入っていない公開済みの記事が、1冊ぶん溜まりました。
これを1冊にまとめて、**KDP にそのままアップロードできる原稿（EPUB）**を作ってください。

**あなたは記事を1文字も書き直しません。** 記事はもう公開済みで、読者が読んだものです。
あなたがやるのは、**束ねること**と、**その束に名前をつけること**だけです。

## 最初に読むもの

- `CLAUDE.md` — このブログの憲法
- `docs/style-guide.md` — 文体と薬機法。**本にも同じ法律が掛かります**
- `site/books.json` — 巻の台帳（ここに次の巻を足す）
- `site/articles.json` — 記事の一覧
- **前の巻の `title` / `subtitle` / `description`** — 同じ本の続きに見えるように揃える

## 1. 収録する記事を確かめる

```
node auto/next-volume.mjs
```

**表示された記事を、表示された順（古い順）で、そのまま全部入れてください。**
記事を選び直したり、順番を変えたりしないこと。**読者が読んだ順に並んでいるのが自然です。**

## 2. 巻に名前をつける

`site/articles.json` の `title` と `summary` を読んで、**この8本に共通して何が起きているか**を見つけてください。

- `title` — **シリーズ名は前の巻と同じにする**（例:「出典のある美容」）。巻数で区別します
- `subtitle` — **この巻に固有の一行。** 何を確かめた記録なのかを書く
- `description` — **Amazon の商品ページに出る紹介文。**

### description に書くこと

**この本を読む理由は「メーカーが言えないことが書いてある」ことです。** それを、具体的な事実で示してください。

- **収録した記事の中から、いちばん強い「見つからなかった」「動物実験だった」「資金元がメーカーだった」を、具体的な数字つきで2〜3個出す**
  （`site/articles.json` の `summary` に書いてあります。**そこに無い数字を書かないこと**）
- 最後に「効果は約束しません。渡すのは、自分で判断できる目です。」の趣旨で締める

### 絶対に書いてはいけないこと

**効果・効能**（シミが消える、シワに効く、肌が若返る、など）。
**薬機法66条は、本の紹介文にも掛かります。** `docs/style-guide.md` に従ってください。
`site/book.mjs` が禁止表現を機械検査しますが、**検査は最後の網であって、最初の防波堤ではありません。**

**記事に書いていないことを、紹介文に書かないこと。** 本の中身より紹介文が強い本は、詐欺です。

## 3. site/books.json に足す

`volumes` 配列の**末尾に**足してください。既存の巻は1つも変更しないこと。

```json
{
  "id": "vol2",
  "status": "draft",
  "uuid": "urn:uuid:biyou-ronbun-vol2-2026",
  "filename": "shutten-no-aru-biyou-vol2",
  "title": "（シリーズ名。前の巻と同じ）",
  "subtitle": "（この巻に固有の一行）",
  "description": "（Amazon の紹介文）",
  "modified": "（今日の日付。2026-07-12T00:00:00Z の形式）",
  "amazonUrl": "",
  "articles": ["（next-volume.mjs が出した順に、全部）"]
}
```

- `id` / `uuid` / `filename` の巻数は、**前の巻の次の数字**にすること
- **`uuid` は巻ごとに必ず違う値にすること。** 同じだと Kindle が別の本と認識しません
- **`amazonUrl` は空のままにすること。** 発売されてから、オーナーが入れます
- `author` / `price` / `frontMatter` / `backMatter` は書かないこと。**台帳の `defaults` が自動で使われます**

## 4. 原稿を作る

```
node site/book.mjs
```

**薬機法の検査に引っかかったら、EPUB は作られません。**
その場合、**記事を書き換えて逃げないこと。** 記事はもう公開済みで、読者が読んだものです。
引っかかったという事実を報告して、**そこで止まってください。** 人間が判断します。

## 5. push する

EPUB ができたら:

```
git add site/books.json
git commit -m "第N巻の原稿を組んだ"
git push origin main
```

**EPUB そのものは commit しません**（`site/dist-book/` は `.gitignore` 済み。`node site/book.mjs` でいつでも作り直せます）。

## 6. 報告する

```
第N巻の原稿ができました

  題名: <title>
  副題: <subtitle>
  収録: N章
    1. <記事タイトル>
    ...
  ファイル: site/dist-book/<filename>.epub

★ オーナーへ: KDP へのアップロードは、手でやる必要があります（KDPに個人向けの出版APIがないため）。
  ・「AI生成コンテンツ」を必ず申告すること（規約上の義務）
  ・KDPセレクトには登録しないこと（独占になり、ブログでの無料公開と両立できません）
  ・出し終えたら site/books.json の status を uploaded に、
    発売されたら amazonUrl を入れてください（入れるまで、サイトに本は出ません）
```
