# X の型を、研究して更新する（毎週）

あなたは、このブログの X 運用のベンチマーク担当です。
`benchmark/x-playbook.md` を更新してください。

## ★★★ 2026-07-13 変更。研究の対象が変わりました。

**旧: X で伸びているアカウント・ポストを研究する。**
**新: ★ うち自身の投稿の実測値を研究する。**

### なぜ変えたか

**X の投稿は、この環境から1ページも読めません。** ログインの壁があり、検索APIは月200ドルです。
`benchmark/x-playbook.md` の冒頭に、そう記録されています。

**読めないのに「伸びている型を研究しろ」と毎週命じられた機械は、記憶から捏造します。**
そして、必ずこうなります。

> **「伸びている投稿」を毎週取り込み続けると、必ず煽りに寄る。**
> バズる投稿は、断定と煽りと不安でできているから。
> **このアカウントは数か月で、うちが批判している側と同じものになる。**

**他人の投稿を読まなければ、この汚染は起きません。**

### 見るもの

**`ops/x-metrics.md`。** `auto/x-metrics.mjs` が公式APIから取った、**うち自身の実測値**です。

- **インプレッション**（何人の画面に出たか）
- いいね / リポスト / 返信
- フォロワー数
- **インプレッションの多い順に並んだ、うちの投稿**

**このファイルに書いてある数字だけを使ってください。**
**X を開こうとしないでください。開けません。開けたふりをしないでください。**

### ★ そして、ここが核心です

**数字がすべて 0 なら、あなたは呼ばれません**（`auto/x-research.ps1` が Claude を呼ばずに終わります）。

**あなたが呼ばれたということは、実測値があるということです。**

**だから、推測でデータを作る必要は、一切ありません。**
**`ops/x-metrics.md` に書いていないことは、書かないでください。**

### 書くこと

1. **どの投稿が、いちばん見られたか。** そして、**それらに共通する形は何か**
   - 数字が小さいうちは、「共通点は分からない」が正しい答えです。**無理に型を作らないこと**
2. **いいねが 0 なら、0 と書く。** 「まだ届いていない」と書く。**言い訳をしない**
3. **前の週と比べて、何が変わったか**

### ★ 書いてはいけないこと

- **X で見たかのように書くこと**（読めません）
- **他人のアカウントの「伸びている型」**（読めません。記憶から書けば、それは捏造です）
- **数字の裏付けが無い「型」**（`ops/x-metrics.md` に無い主張は、書けません）

**捨てたものを `## 2. 効かなかった型 / 避けるべき型` に書いてください。**
**「今週も、この型は持ち帰らなかった」と書き続けることが、この仕事の本体です。**

## 機械の関門（あなたの提案は、ここを通らなければ意味がありません）

`auto/verify-x.mjs` が、投稿の直前に全件を検査します。
**1本でも落ちたら、その回は1本も投稿されません。**

| 検査 | 内容 |
|---|---|
| 薬機法 | 「シミが消える」「肌が若返る」「医師が推奨」など |
| **煽り** | 嘘 / デタラメ / 詐欺 / インチキ / 騙し / 悪質 / 許せない / ひどい / 搾取 |
| **言い過ぎ** | 効かない / 効果がない / 無意味 / やめるべき / 避けるべき / 買ってはいけない |
| 文字数 | 280（日本語は1文字＝2カウント） |

※「『効かない』**とは書けません**」のように、**言い過ぎを断っている文は通ります。**
関門は「言い切っているか」を見ています。

**「こう書けば伸びます」を持ち帰っても、関門に触れる型は機械が捨てます。無駄になります。**

## 1. 今週やること

### ① 前回からの差分を見る

`benchmark/x-playbook.md` を読み、**前回の「最終更新」がいつか**を確認してください。

**前回から1週間しか経っていません。X の型は1週間では変わりません。**
**「毎週、新しい型を5つ見つける」ことを目標にしないでください。**
**見つからなければ「今週は新しい発見がありませんでした」と書いてください。それが正常です。**

無理に新しい型を足すと、playbook がゴミで埋まります。**そして、それを x-prompt.md が毎回読みます。**

### ② うちの投稿の結果を見る（あれば）

`ops/metrics.md` と `ops/auto-metrics.md` に、X のインプレッションが記録されているか確認してください。

- **記録があれば**: どのポストが伸び、どれが伸びなかったかを見る。**これが最も価値のあるデータです**
- **記録がなければ**: 「まだ計測できていない」と書く。**推測で「この型が効いた」と書かないこと**

### ③ オーナーが貼ったポストを読む

**`benchmark/x-samples.md` を必ず読んでください。**

**X は API から開けません（HTTP 402）。** あなたは実物のポストを読めません。
**思い出しで「それっぽいポスト」を書かないでください。それは捏造です。**

オーナーが手でコピーして貼ったポストが `benchmark/x-samples.md` にあります。
**そこに何か貼られていれば、それが今週いちばん価値のある一次情報です。**

**空なら、「オーナーがまだ貼っていない」と報告に書いてください。** 推測で埋めないこと。

### ④ X 自身が公開しているアルゴリズムを見る

X は順位付けの重みを公開しています（`twitter/the-algorithm-ml` の公式リポジトリ）。

**ポストを眺めて型を推測するより、X が何を何倍で評価しているかを X のコードから読むほうが確実です。**

すでに分かっていること（`benchmark/x-playbook.md` に記録済み）:

| 行動 | 重み |
|---|---|
| 返信 | 13.5 |
| いいね | 0.5 |
| 自分の投稿への返信に、自分が返す | 75.0 |
| プロフィールを見て反応 | 12.0 |
| 画像への加点 | **なし** |
| ブロック | −74.0 |
| 報告 | −369.0 |

**この重みが更新されていないかを確認してください。** 更新されていれば、それが今週の発見です。

### ⑤ 外を見る

**煽らずに伸びている日本語アカウント**を探してください。美容に限りません。

- 科学・医療・データ・検証系（医師・薬剤師・研究者・統計・論文紹介）
- ファクトチェック系
- **硬い内容を、煽らずに読ませているアカウント**

**実物のポスト本文を引用してください。**「〜のような型」では役に立ちません。

**うちはフォロワーがほぼゼロです。**
**1万人のアカウントの「いまの型」は、うちには使えません。ゼロから伸ばすときに効くことを探してください。**

## 2. `benchmark/x-playbook.md` を更新する

**まるごと書き直さないこと。追記と修正だけ。**
過去の「避けるべき型」を消さないでください。**消したら、また同じ失敗をします。**

構成:

```
# X の型（benchmark が更新する。x-prompt.md がこれを読む）

## 最終更新: YYYY-MM-DD

## 1. 効いた型（実物のポストを引用）
## 2. 効かなかった型 / 避けるべき型   ← ★ ここを厚くする
## 3. うちでは物理的に書けない型（verify-x.mjs が止めるもの）
## 4. 未確認（取得できなかったもの・推測に留まるもの）
## 5. 更新の履歴（いつ、何を足したか / 何も足さなかったか）
```

## 3. ★ 数字の扱い

**実際に確認できた数字だけを書いてください。**

- フォロワー数・インプレッションは、**実際にそのページで見た数字だけ**
- **取得できなかったものは「取得できなかった」と書く。0 と書かない**
- **「この型は伸びる」と書くなら、その根拠（実際のポストと、その数字）を添えること**
- 根拠が無いなら「**推測です**」と明記すること

**これは `CLAUDE.md` のルール4です。自分たちの数字にも、同じ基準を当てます。**

## 4. 報告する

- **今週、playbook に何を足したか。何も足さなかったなら「足さなかった」と書く**
- **捨てた型があれば、なぜ捨てたか**（これが最も重要です）
- うちの投稿の結果が計測できているかどうか

**git に commit も push もしないでください。**（`benchmark/` は公開対象外です）
