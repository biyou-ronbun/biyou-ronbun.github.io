# 海外の学術誌の新着を見て、ネタ帳を更新する（毎週）

**`ops/journal-watch.md` に、海外10誌の新着が書いてあります。まず、それを読んでください。**

---

## ★★★ あなたが絶対にやってはいけないこと

### ✗ 新着を、翻訳して、要約して、記事にする

**それは「他人の情報を、右から左に流す」ことです。**

Google のスパムポリシーは「**AIで価値を加えないページの大量生成**」を明示的に禁じています（自動か人力かを問わない）。

**このサイトを殺すのは、書かないことではなく、薄いものを書くことです。**

**実際に、AIで大量生成したサイトが検索から丸ごと消えた事例があります。**

### ✗ `articles/` を触る

**この輪は、記事を書いてはいけません。**

**触ったら、`auto/journal.ps1` が git で元に戻します。** そして、その週は失敗として記録されます。

### ✗ 「要約」をする

**うちがやっているのは「要約」ではありません。**

**「一次情報まで辿って、ここまでは言えませんと書くこと」です。**

## ★ あなたがやること

**新着の中から、「RCT / メタ解析 / システマティックレビュー / 撤回」だけを見ます。**

そして、次の3つに仕分けます。

### ① うちの既存記事に、直接ぶつかるもの

**例:** 飲むコラーゲンの新しいメタ解析が出た → うちの `articles/oral-collagen.md` の結論に関わる

**このときやること:**

1. **`node site/pubmed.mjs get <PMID>` で、一次情報を開く**
   （★ 記憶から書かない。**開いたページに書いてあったことだけ**）
2. **その記事の錨を確認する**
   `node site/set-anchors.mjs <slug> --show`
3. **`ops/journal-watch.md` に、「この記事に影響する」と追記する**
4. **★ 記事は書き換えない。** オーナーに報告するところまで

**なぜ書き換えないか:**

記事の更新は、**錨を確認し、`check-shorten.mjs` で検算し、公開ページで実物を見るまで**が1セットです。
**この輪は、そこまでできません。記事を触った瞬間、事実が消えます。**

### ② 新しいネタになりそうなもの

**例:** バクチオール（レチノールの代替とされる成分）のRCTが出た → うちはまだ書いていない

**このときやること:**

1. **`topics.json` の `topics` に、`status: "queued"` で追加する**
   - `slug`（既存と重複しないこと）
   - `title`（読者の言葉で。**答えが分かる形**が望ましい）
   - `note`（この論文が出たこと、PMID）
2. **★ 記事は書かない。** 記事の輪（`auto/run.ps1`）が、いつもの流れで作ります

### ③ どちらでもないもの

**何もしない。** `ops/journal-watch.md` に残っているので、それで十分です。

## ★★ 「今週は何も無かった」が、最も多い正解です

**美容の一次研究が、毎週たくさん出ることはありません。**

**「せっかく呼ばれたのだから、何か見つけなければ」と思った瞬間、**
**あなたは、関係のない論文を無理やり結びつけ始めます。**

**そして、記事を1本書きたくなります。書かないでください。**
